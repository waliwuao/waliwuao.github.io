Introduction to Theory of Computation - Lecture 1

发表于 2026-03-03 分类于 课堂笔记

Course Overview

The course is structured into two fundamental pillars that define the limits and efficiency of mathematical logic and machine processing.

Part I: Automata & Computability

  • Boundaries of Computation: Defining which problems are theoretically “computable” and which are not.
  • Theoretical Framework: Understanding the fundamental limits of logic and machines.
  • Computability vs. Feasibility: Distinguishing between problems that can be solved in principle versus those that cannot be solved within finite or reasonable resources (time and space).

Part II: Computational Complexity

  • Efficiency Analysis: Developing a rigorous understanding of what constitutes “computational efficiency.”
  • Tractability: Identifying problems that are solvable using realistic and limited resources.
  • Complexity Hierarchy: Investigating the inherent difficulty of problems and what makes certain tasks fundamentally harder than others.

Lecture 1: Foundations

FSA(Finite States Automata)

A finite automaton $M$ is defined as a 5-tuple $(Q, \Sigma, \delta, q_0, F)$ consisting of:

  • $Q$ : A finite set of states.
  • $\Sigma$ : A finite set of alphabet symbols.
  • $\delta$ : The transition function, defined as $\delta: Q \times \Sigma \to Q$.
    • The notation $\delta(q, a) = r$ indicates that when the machine is in state $q$ and receives input symbol $a$, it moves to state $r$.
  • $q_0$ : The start state, where $q_0 \in Q$.
  • $F$ : The set of accept states, where $F \subseteq Q$.
